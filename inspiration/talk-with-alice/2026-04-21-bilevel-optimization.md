# 嵌套优化：为什么 LLEE 需要两层算法

> 2026-04-21
> 回应：计划成熟度评审中关于"离散参数可微，是否需要改变遗传算法"的问题

---

## 一、问题的本质

LLEE 的可优化参数分成两类，它们有根本不同的数学性质：

```
离散结构（不可微，需要组合搜索）：
  - 证据类型的数量和定义（6 种够不够？）
  - 枚举值集合（scene_type 有哪些合法值？）
  - Schema 字段的存在性（需不需要 scene_features？）
  - 证据类型之间的层级关系

连续参数（可微，可以用梯度）：
  - 衰减率：0.7, 0.9, 1.0
  - 置信度上限：0.30, 0.50, 0.85, 0.90
  - source_weight：0.7, 0.5, 0.8
  - render_intensity 映射曲线的阈值和斜率
  - 贝叶斯先验分布参数
```

用遗传算法优化连续参数，就像用锤子拧螺丝——能做，但效率极低。遗传算法在连续空间中的搜索是随机的，而梯度下降沿着损失函数的梯度方向走，每一步都是"最优的局部移动"。

反过来，用梯度下降搜索离散结构也不行——"要不要增加一个新的证据类型"不是一个可微的决策。

---

## 二、嵌套优化（Bi-level Optimization）

解法是把两层分开：

```
外层（进化算法）：搜索离散结构
  输入：Schema 拓扑的种群（20 个变体）
  操作：增删证据类型、拆分/合并枚举、调整字段存在性
  输出：每个变体的适应度分数

  内层（梯度下降）：对每个变体，优化连续参数
    输入：固定的 Schema 拓扑 + Phase 3 人类评分数据
    操作：梯度下降优化衰减率、上限、映射曲线
    输出：该拓扑下的最优损失值 → 作为外层的适应度

适应度 = 内层优化后的最佳损失值
```

这样每个 Schema 变体都在"最优参数"下被评估。如果变体 A 的结构比变体 B 好，但参数没调好，纯遗传算法可能错误地淘汰 A。嵌套优化避免了这个问题。

---

## 三、这和 LLEE 的理论框架有什么关系？

嵌套优化的两层，恰好对应 LLEE 的两个核心概念：

```
外层（离散结构搜索）↔ "符号骨架"
  - 证据类型的定义是符号系统的语法
  - 进化算法搜索的是"什么样的语法最适合描述叙事世界"
  - 这是 LLEE 论文中"可进化的符号系统"的直接实现

内层（连续参数优化）↔ "参数血肉"
  - 衰减率、上限、映射曲线是附着在骨架上的参数
  - 梯度下降优化的是"在给定语法下，什么参数最能匹配人类感知"
  - 这是 Scheme C（内部连续，外部离散）的自然延伸
```

LLEE-environment.txt 中提到的"符号骨架 + 神经参数的混合系统"——嵌套优化就是这个混合系统的算法实现。

## 四、和 NAS（Neural Architecture Search）的类比

嵌套优化在深度学习中有一个成熟的对应物：**神经架构搜索（NAS）**。

```
NAS：
  外层：搜索网络架构（层数、宽度、连接方式）— 离散
  内层：训练网络权重 — 连续
  适应度：验证集准确率

LLEE 嵌套优化：
  外层：搜索 Schema 拓扑（证据类型、枚举集合）— 离散
  内层：优化 Schema 参数（衰减率、上限、映射曲线）— 连续
  适应度：人类评分 + CLIP + 归一化忠实度
```

NAS 的经验教训对 LLEE 有直接参考价值：

1. **权重共享（weight sharing）**：NAS 中，不同架构共享部分权重以加速评估。LLEE 中，不同 Schema 变体共享相同的 Parser prompt 和 WSM 逻辑，只有参数不同——这天然就是权重共享。

2. **超网络（supernet）**：NAS 中，所有候选架构被编码为一个超网络的子图。LLEE 中，可以定义一个"超 Schema"，包含所有可能的证据类型和字段，每个变体是超 Schema 的一个子集。

3. **早停（early stopping）**：NAS 中，明显差的架构在训练早期就被淘汰。LLEE 中，如果一个 Schema 变体在 5 轮梯度下降后损失仍然很高，可以提前淘汰，节省计算资源。

## 五、早熟收敛的对策

评审中提到的"种群多样性"问题，在嵌套优化中更容易解决：

```
多样性维护机制：
  1. Jaccard 相似度监控：
     每代计算种群中 Schema 的 Jaccard 相似度
     如果平均相似度 > 0.8 → 触发移民

  2. 移民策略：
     随机生成 3 个全新 Schema（不基于现有种群）
     替换种群中适应度最低的 3 个个体
     这保证了每代都有"新鲜血液"

  3. 岛屿模型（可选）：
     将种群分成 4 个子群（岛屿），各自独立进化
     每 10 代交换 1 个最优个体
     不同岛屿可能收敛到不同的局部最优 → 增加全局探索
```

## 六、对计划的影响

Phase 4 从"纯遗传算法"升级为"嵌套优化"，工程复杂度增加不大（内层梯度下降只需要 PyTorch 的自动微分），但理论贡献显著提升：

- 纯遗传算法的论文叙事："我们用进化搜索了更好的 Schema"
- 嵌套优化的论文叙事："我们证明了 LLEE 的符号结构和连续参数可以被联合优化，且两层优化的贡献可以被独立量化"

后者更接近 NeurIPS/ICML 的审稿标准。

## 七、一个更远的联想

嵌套优化的外层搜索离散结构、内层优化连续参数——这和人类语言的演化有惊人的相似：

```
人类语言演化：
  外层（代际时间尺度）：语法结构的变化（SVO → SOV，屈折 → 分析）
  内层（个体时间尺度）：词汇和用法的微调（新词、语义漂移）

LLEE 演化：
  外层（进化代数）：Schema 拓扑的变化（新增/删除证据类型）
  内层（梯度步数）：参数的微调（衰减率、上限）
```

如果 LLEE 的嵌套优化在 Phase 4 中成功，它不仅证明了"Schema 可以自动优化"，还暗示了一个更深的命题：**符号系统的演化天然具有两个时间尺度——结构的慢变化和参数的快适应。** 这和语言学中的"语法化（grammaticalization）"理论是同一个模式。
