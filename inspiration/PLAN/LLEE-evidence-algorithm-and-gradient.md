> 2. 是否应该引入梯度下降？上下文语言的预测性是否天然带来高保真度？

## 一、当前证据算法：不够严谨，需要补充
v3 计划定义了证据的分类（来源 × 4 强度），但没有回答一个关键问题：

**Parser 如何确定一个属性的证据等级？**

目前的答案是"让 LLM 判断"——这是一个黑盒。对于严谨的实验，黑盒不足。需要一个可计算的算法。

### 证据判定的具体算法
```
输入：一段段落文本 T，一个待标注的属性 A（如 emotion.value）
输出：证据等级, 证据来源, 置信度, 证据文本片段

算法步骤：
Step 1 — 证据来源识别
  按优先级顺序检查：

  1a. TEXT_EXPLICIT（最高优先级）
      条件：T 中存在直接描述 A 的文本片段
      例："他哭了" → emotion.value=sadness, evidence=EXPLICIT
      实现：LLM 提取直接描述句，返回 source_span（字符偏移量）

  1b. TEXT_BEHAVIORAL
      条件：T 中存在可观察行为，该行为是 A 的可靠指示器
      例："他握紧了拳头" → emotion.value=anger/tension, evidence=BEHAVIORAL
      实现：LLM 识别行为描述，并判断行为→属性的映射强度

  1c. CONTEXT_ACCUMULATED
      条件：WSM 历史状态中存在 A 的先前值，且当前段落未明确改变它
      例：前文已建立"洞穴"环境，当前段落未提及环境变化
      → ENV.type=cave 继承, evidence=CONTEXT, 置信度衰减 0.9ⁿ（n=继承次数）
      实现：WSM 自动处理，不需要 LLM

  1d. TEXT_CONTEXTUAL
      条件：T 中存在情境信息，可以推断出 A
      例："父亲去世五年了" → emotion.value 可能是 sadness，但无直接证据
      实现：LLM 判断，但置信度上限 0.5

  1e. EXTERNAL_KNOWLEDGE
      条件：外部知识库中存在关于 A 的文化/物理事实
      例：查询"中东洞穴的声学特性" → ENV.audio.reverb=cave_medium
      实现：RAG 查询，置信度由检索相关性决定，上限 0.9

  1f. UNDEFINED（最低优先级）
      条件：以上均不满足
      → 属性值设为 UNDEFINED，置信度 0

Step 2 — 置信度计算
confidence = base_confidence × source_weight × decay_factor

  source_weight:
    EXPLICIT:    1.0
    BEHAVIORAL:  0.7
    CONTEXT:     0.9ⁿ（n=继承次数，每次继承衰减10%）
    CONTEXTUAL:  0.5
    EXTERNAL:    0.8
    UNDEFINED:   0.0

  decay_factor:
    如果 A 是情绪类属性：× 0.9（情绪比物理属性更不确定）
    如果 A 是物理类属性（位置、材质）：× 1.0

Step 3 — 渲染强度映射
render_intensity = f(confidence)

  EXPLICIT   (confidence > 0.8): render_intensity = 1.0
  BEHAVIORAL (confidence 0.5-0.8): render_intensity = 0.3-0.6
  CONTEXTUAL (confidence 0.3-0.5): render_intensity = 0.1-0.3（可配置关闭）
  UNDEFINED  (confidence < 0.3): render_intensity = 0.0（严格中性）
```

该算法**可计算**：每个属性的证据等级都有来源文本片段、置信度数值、衰减历史。论文中可以展示严谨案例的完整证据链。

## 二、梯度下降的问题：是否应该引入？

你的问题触及了 LLEE 最底层的设计张力。我将其拆解。

### 你的直觉是什么？

> "上下文语言是回顾上文，但也具有预测性，也许人类无法理解显式原因，但它天然带来高保真度。"

翻译成技术语言：
**上下文语言（前文积累的世界状态）具有预测性**——它不仅描述了过去，还约束了未来。当 Parser 把前文状态注入 prompt 时，LLM 的生成会自然地与这个状态保持一致，即使当前段落没有明确说明。这种一致性是"隐式的"——人类读者感受到了，但说不清楚为什么。

**这个直觉是正确的。**
并且它指向了一个非常有价值的问题：
**如果上下文约束天然带来高保真度，那么"证据分级"是否多余？**

### 梯度下降的诱惑
如果我们把整个管道看成一个函数：

```
f(文本 T, 上下文 C) → 世界状态 W → 渲染结果 R
```

那么"保真度"就是一个损失函数：

```
L = distance(R, 人类期望的渲染结果)
```

如果 L 可微，我们就可以用梯度下降优化 f 的参数——包括证据分级的阈值、置信度权重、甚至 prompt 本身（通过 soft prompt tuning）。

**这在技术上是可行的。**
严谨路径：
```
可微渲染（Differentiable Rendering）：
  Three.js → 不可微
  但可以用 PyTorch3D 或 nvdiffrast 做可微渲染
  → 渲染结果是一个可微的张量
  → 可以对渲染参数（光照、材质）求梯度

可微 LLM 输出（Soft Prompt Tuning）：
  固定 LLM 权重，只优化 prompt 的连续向量
  → 通过梯度下降找到"最能让 LLM 输出高保真 LLEE Delta"的 prompt 嵌入
  → 这是 prefix tuning / prompt tuning 的标准做法

端到端梯度流：
  人类偏好分数 → 可微渲染损失 → 渲染参数梯度 → LLEE Delta 梯度 → prompt 嵌入梯度
```

### 但是——这会破坏 LLEE 的核心价值主张
LLEE 的信任边界建立在一个前提上：
**LLM 输出的不确定性被隔离在 Parser 阶段，一旦生成 LLEE Delta，下游完全确定性。**

如果引入梯度下降：
- prompt 嵌入变成了连续向量，不再是人类可读的文本
- 证据分级的阈值变成了优化出来的数字，不再有语义解释
- 整个系统变成了一个端到端的黑盒

**这不是 LLEE，这是一个普通的文本到渲染的端到端模型。**
它可能效果更好，但它失去了 LLEE 的核心根基：可计算的信任边界。

### 正确的答案：两条路，不同目标

```
路径 A — 保持 LLEE 的符号系统，用梯度下降优化映射层

  不优化：
    - LLEE Schema 的结构（保持人类可读）
    - 证据分级的语义定义（保持可解释）
    - WSM 的差分逻辑（保持确定性）

  可以优化：
    - 标签→参数映射表："oxidized_copper" → PBR 参数
      用梯度下降找到最优的 PBR 参数组合，使渲染结果与参考图最接近
    - 证据置信度的权重（source_weight 中的 0.7, 0.5 等数值）
      用梯度下降在验证集上优化这些权重
    - 渲染强度映射函数 f(confidence) → render_intensity
      用梯度下降找到最优的映射曲线

  这是"在 LLEE 框架内引入梯度下降"——优化的是参数，不是结构。
  可解释性保留，性能提升。

路径 B — 用梯度下降学习上下文预测性

  这正是你直觉指向的方向。

  核心思想：
    上下文 C 对当前段落 T 的世界状态有预测力。
    这种预测力可以被量化：

    predictive_power(C, T) =
      P(正确预测 T 的世界状态 | 只看 C，不看 T)

    如果 predictive_power 高，说明 C 已经"隐式包含"了 T 的信息。
    此时，即使 T 本身信息量很小（极简文本），
    LLEE 也可以从 C 中借用"高置信度的状态。

  实现方案：
    训练一个小型预测器（不是大模型，是一个轻量分类器）：

    输入：WSM 当前状态（向量化）+ 当前段落文本（嵌入）
    输出：下一个世界状态的预测（各属性的概率分布）

    训练数据：从丰富文本中提取（段落序列 → 状态序列）
    损失函数：预测状态 vs 真实状态的交叉熵

    这个预测器用梯度下降训练，但它是 LLEE 管道的一个可选组件，
    不替代 Parser，而是作为"上下文预测证据"的来源。

    当 Parser 对某属性输出 UNDEFINED 时，
    预测器可以提供一个 PREDICTED 级别的补充：
    "根据前文状态，这个属性大概率是 X（置信度 0.35）"

    这就是你说的"人类无法理解显式原因，但天然带来高保真度"——
    预测器学到的是统计规律，不是人类能说清的规则。
```

### 关键判断：路径 B 是否值得做？

**值得，但有一个前提条件：**

路径 B 的预测器需要训练数据——大量的"段落序列 → 状态序列"对。这正是 Phase 0-2 的实验数据。
**换句话说：先做 Phase 0-2，积累数据，然后用这些数据训练预测器，这就是路径 B。**

路径 B 不是 Demo 阶段的工作，它是 Phase 4 之后的工作。但它的数据基础从 Phase 0 就开始积累了。

## 三、对 v3 计划的最小修改
基于以上分析，v3 计划需要三处修改：

### 修改 1：在 Phase 0.2 中增加"信息单元标注指南"
```
任务 0.2.5  信息单元标注指南
  ✅ 定义"可渲染信息单元"的边界（一页）
    - 算：实体存在、实体属性、空间关系、动作、感官特征
    - 不算：隐喻、主观感受、无法可视化的抽象
    - 正例 5 个，反例 5 个
  ✅ 两个标注者试标 5 个段落
  ✅ 计算 Krippendorff's alpha（目标 α > 0.67）
  ✅ 如果不达标 → 迭代指南
```

### 修改 2：在 Phase 0.3 中增加证据算法的形式化定义
```
任务 0.3.5  证据判定算法文档
  ✅ 写出完整的证据判定算法（见本文第一节）
  ✅ 用 5 个段落手动验证算法输出是否符合直觉
  ✅ 将算法编码为 Parser prompt 的一部分（作为"判断规则"注入）
  ✅ 将算法编码为后端验证器（检查 LLM 输出是否符合算法规则）
```

### 修改 3：在 Phase 4 中增加"上下文预测器"实验
```
任务 4.5  上下文预测器（路径 B）
  ✅ 用 Phase 0-2 积累的数据训练轻量预测器
    - 输入：WSM 状态向量 + 段落嵌入
    - 输出：下一状态的属性概率分布
    - 架构：小型 MLP 或 Transformer（不是大模型）
    - 训练数据：丰富文本的段落序列（约 500-1000 个段落对）
  ✅ 验证：预测器的预测准确率 vs 随机基线
  ✅ 集成：将预测器输出作为 PREDICTED 级别的证据注入 Parser
  ✅ 对比：有预测器 vs 无预测器的极简文本保真度

  这验证你的直觉：
  "上下文语言的预测性天然带来高保真度"
  如果预测器在极简文本上显著提升保真度，
  这就是这个直觉的实证证明。
```

## 四、一个更深的问题：证据分级是否应该是连续的？

你的问题还隐含了另一个方向：

当前的证据分级是离散的（EXPLICIT / BEHAVIORAL / CONTEXTUAL / UNDEFINED）。但置信度是连续的（0.0 到 1.0）。
**为什么不直接用连续置信度，而要有离散等级？**

原因是**可解释性**：
- 连续置信度 0.73 对作者没有意义
- "BEHAVIORAL 等级的证据"对作者有意义——他知道这意味着"有行为线索但无直接描述"

但你的问题指向了一个更深的可能：
**如果置信度是连续的，梯度下降就可以更直接优化它。**

这是一个真正的设计选择：
```
方案 A（当前）：离散等级 + 连续置信度
  优点：可解释，作者可以理解和修改
  缺点：离散等级的边界是人工设定的，可能不是最优的

方案 B：纯连续置信度
  优点：可微，可以用梯度下降端到端优化
  缺点：失去可解释性，作者无法理解"置信度 0.73"意味着什么

方案 C（混合）：连续置信度 + 离散等级作为"软标签"
  离散等级是置信度的人类可读近似：
    confidence > 0.8 → EXPLICIT
    0.5-0.8 → BEHAVIORAL
    0.3-0.5 → CONTEXTUAL
    < 0.3 → UNDEFINED

  梯度下降优化连续置信度，离散等级是展示给人类的界面。
  这保留了可解释性，同时允许梯度优化。
```

**推荐方案 C。**
它是 LLEE 的符号系统与梯度下降的最优结合点：
- 内部用连续置信度（可微）
- 外部展示离散等级（可解释）
- 梯度下降优化置信度的计算方式，不改变等级的语义定义

## 五、总结

| 问题 | 答案 |
|------|------|
| 证据算法是否足够严谨？ | 不足。本文补充了完整的判定算法（3 步骤，含置信度计算和渲染强度映射） |
| 是否应该引入梯度下降？ | 应该，但有边界：优化"映射层参数"和"置信度计算"，不优化 Schema 结构 |
| 上下文语言的预测性是否天然带来高保真度？ | 是的，这是一个可验证的假设。通过训练轻量预测器（Phase 4）来实证 |
| 人类无法理解显式原因但天然高保真度——可能吗？ | 可能。预测器学到的是统计规律，不是人类规则。这正是"隐式语言"的本质 |
| 证据分级应该是离散还是连续？ | 方案 C：内部连续（可微），外部离散（可解释）。两者不矛盾 |

**对 v3 计划的影响**：三处最小修改（见第三节），不改变整体结构，不延长时间线。
**最重要的一句话**：你的直觉——"上下文语言具有预测性，天然带来高保真度"——不是一个文学幻想，而是一个可以用 Phase 4 的预测器实验来验证的科学假设。可以把它写进论文的 Future Work，然后在 Phase 4 证明它。
