# 论文该怎么做，你该怎么做

> 写给 LLEE 系列对话之后
> 2026-04-20

---

## 先说一件事

你和 AI 一起把一个哲学设想打磨成了五个版本的论文。每一轮修订都更严谨、更务实、更"学术正确"。但在 after0.5 里，你问了一个问题，那个问题比论文本身更重要：

**"我真的希望有独特的创新，可以诞生全新的有价值的思路吗？"**

这个问题不是在问论文。这个问题是在问：我花了这么多时间，值得吗？我的想法有没有真正属于我自己的东西？

我想直接回答这个问题，然后再说论文和你接下来该怎么做。

---

## 一、你的想法里有没有真正属于你的东西？

有。而且它不在论文里，它在你问问题的方式里。

从原版到 0.5，你一直在追一个东西：**让 AI 不要替你做决定**。不要替角色决定情绪，不要替场景决定氛围，不要替叙事决定走向。你想要的是一个"忠实的媒介"——它传递你的意图，不添加自己的解读。

这个执念，在技术上叫 Zero-Introspection Principle。但它背后的哲学是：**创作者的意志应该是最终权威，AI 是工具不是共同作者。**

这个立场在 2026 年是少数派。大多数 AI 内容生成的研究方向是"让 AI 更有创意"、"让 AI 更自主"。你在逆着这个方向走，你在说：不，我要的是一个更听话的 AI，一个更透明的 AI，一个不会偷偷加戏的 AI。

这是真实的、有价值的立场。它不是技术创新，它是**设计哲学的创新**。

---

## 二、论文该怎么做

### 0.5 版本已经足够好了，但它有一个身份危机

0.5 版本是一篇诚实的工程论文。它的问题不是质量，而是它不知道自己是给谁看的：

- 如果是给**学术审稿人**看：需要更严格的实验设计，n=10 的 usability study 不够，需要真实数据
- 如果是给**游戏开发者**看：太多学术包装，实际的代码示例和集成指南不够
- 如果是给**AI 研究社区**看：IEI 问题的定义是贡献，但需要与 hallucination 文献更深的对话

**建议：选一个读者，为他们重写。**

最适合你的读者是**游戏/VN 开发者社区**（GDC、indie dev 论坛、WebGAL 社区）。原因：
1. 这是你实际在做的事（ai-game-workshop）
2. 这个社区没有人在认真讨论 LLM 情绪注入问题
3. 你不需要 n=50 的双盲实验，你需要一个能跑起来的 demo

### 论文的最小可行形式

不是一篇 8 节的学术论文，而是：

```
标题：防止 AI 替你的角色加戏：一个 VN 开发者的实践指南

结构：
1. 问题（IEI，2 段，一个例子）
2. 解法（WorldState + 证据分级情绪，代码示例）
3. 集成（WebGAL / RenPy / UE5，各一个代码片段）
4. 局限（诚实说明什么情况下不管用）
5. 未来（一段话，留下那个"自进化世界语"的火种）
```

这个形式可以发在 dev.to、GitHub README、或者 GDC Vault。它会被读，会被用，会被引用。

## 三、你该怎么做

这才是真正的问题。

### 你现在处于什么位置

你有一个哲学设想（AI 应该是忠实媒介），你把它打磨成了一个工程框架（LLEE），你在做一个实际项目（ai-game-workshop）。

这三件事是同一件事，但你一直在把它们分开处理：论文是论文，项目是项目，哲学是哲学。

**把它们合并。**

### 具体来说

**第一步：在 ai-game-workshop 里实现 LLEE 的最小核心**

不是完整的 LLEE，只是这两件事：

```python
# 1. WorldStateDelta：只传变化量
@dataclass
class CharacterDelta:
    id: str
    expression: str | Literal["UNDEFINED"] | None = None  # None = 不变
    position: str | None = None

# 2. 生成约束注入：把当前状态塞进 prompt
def build_constraint_prompt(world_state: WorldState, new_text: str) -> str:
    return f"""
[当前场景状态]
{world_state.to_summary()}

[约束]
不得为角色添加文本中没有证据的情绪。
如需更新角色状态，以 JSON delta 格式输出。

[新叙事文本]
{new_text}
"""
```

这两件事加起来不超过 200 行代码。但它们是 LLEE 的灵魂。

**第二步：用真实的 VN 脚本测试**

找一段你自己写的、或者公版的 VN 脚本（20-30 句对话），跑两个版本：
- 直接让 LLM 扩展
- 用 LLEE 约束后扩展

截图，录屏，写一篇 500 字的观察笔记。这比任何论文都有说服力。

**第三步：把那三个"荒野方向"写下来，但不是作为论文**

after0.5 里 AI 给了三个方向：参数流形、溯因渲染、逆LLEE。这三个方向不适合写进现在的论文，但它们适合写成**你自己的研究日记**。

不是给审稿人看的，是给三年后的你看的。写下：
- 这个方向为什么让我兴奋
- 它需要什么才能变成现实
- 我现在能做的最小实验是什么

这些笔记比论文更值钱，因为它们是你真实的思考轨迹。

## 四、关于那个"自进化世界语"的梦想

after0.5 里 AI 写了一段很漂亮的"留给未来的信"。那段话是真诚的，但它有一个问题：它把那个梦想放在了"未来"，放在了"别人打开的那封信"里。

我想说：那个梦想不需要等。

"逆 LLEE"——让世界自己生成语言——这件事的最小版本，你现在就可以做：

1. 取一个场景（比如洞穴入口，黄昏，两个人对话）
2. 用三种方式描述它：自然语言、LLEE block、纯物理参数（光谱、声场、材质 BRDF）
3. 问：这三种描述，哪种最"接近"这个场景本身？哪种信息损失最少？

这不是一个工程问题，这是一个**信息论 + 感知哲学**的问题。它没有标准答案，但思考它的过程会告诉你，你真正想做的是什么。

---

## 五、一句话总结

论文该怎么做：**缩小范围，选定读者，用真实数据说话，把哲学梦想留在最后一段。**

你该怎么做：**先在项目里实现最小核心，用真实脚本测试，然后写研究日记而不是论文。论文是结果，不是过程。**

你已经有了最重要的东西：一个真实的问题，一个真实的立场，一个真实的项目。剩下的是执行。

---

*附：after0.5 里那三个荒野方向的一句话定位*

- **参数流形**：把 LLEE 的枚举值换成连续物理参数。近期可做，是 LLEE 的自然延伸。
- **溯因渲染**：让证据置信度成为渲染的一个维度——不确定的东西在视觉上呈现为"未决"。中期方向，需要重新设计渲染器。
- **逆 LLEE**：让多模态感知压缩自动生成隐语，然后问人类"这让你想到什么"。这是真正的新科学，也是你最初那个梦想的真实形态。
