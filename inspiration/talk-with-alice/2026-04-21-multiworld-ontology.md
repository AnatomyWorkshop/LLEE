# 架空世界的置信度与多世界本体论

> 2026-04-21
> 核心问题：架空世界的置信度是否应该和真实世界相同？
> 多世界之间如何交换证据？

---

## 一、置信度衡量的是什么

LLEE 的 confidence 不是"物理真实性"，是"文本证据强度"。

```
"巨大的精灵从地下升起"  → EXPLICIT, confidence 0.9
  理由：叙述者直接陈述。精灵在物理世界不存在，但在叙事世界中是事实。

"洞穴很冷"              → CONTEXTUAL, confidence 0.4
  理由：文本没说温度，但洞穴通常冷。这是推断，不是事实。

"Aladdin 害怕"          → UNDEFINED
  理由：文本没说他害怕。Zero-Introspection 原则禁止推断。
```

三者的置信度差异不是因为"精灵比温度更真实"，而是因为"文本对精灵的描述比对温度的描述更明确"。

**这意味着：架空世界和真实世界在 LLEE 框架内完全平级。**

一个精灵的 EXPLICIT 证据和一把椅子的 EXPLICIT 证据，在渲染管道中享有完全相同的权重。置信度低不意味着"更幻想"，意味着"文本证据更弱"。一个在现实世界中完全可能的事件（"他可能饿了"），如果文本没有明确说，置信度就是低的。一个在物理上不可能的事件（"精灵出现了"），如果文本明确说了，置信度就是高的。

**LLEE 的 ground truth 是文本，不是物理现实。**

---

## 二、对评估指标的影响

这个认识暴露了 CLIP 分数作为评估指标的一个隐含偏差：

CLIP 是在真实图像上训练的。它对"一个人站在洞穴里"的文本-图像对齐分数，可能高于"巨大的精灵从地下升起"——不是因为后者的渲染不忠实，而是因为 CLIP 的训练数据中精灵图像少。

**归一化忠实度没有这个问题。** 它只看"文本说了什么，渲染了什么"，不依赖外部的视觉先验。

这进一步支持了计划中"双度量体系"的设计：CLIP 作为自动化筛选工具（有偏但可复现），归一化忠实度作为公平度量（无偏但需要人工标注）。

## 三、多世界本体论

Karl 的文本是一个完美的案例。他有预知能力，能看到多个可能的未来：

```
主世界 W₀：教室，Karl 站着，被要求自我介绍
  ├── 幻象 W₁：说 Fortnite → 社交死亡
  ├── 幻象 W₂：说冲浪 → 被嘲笑
  └── 幻象 W₃：说烧玫瑰 → 成功
      └── W₃ 的台词被 Karl 采用，反馈到 W₀
```

每个世界内部的置信度规则完全相同。W₁ 里"同学嘲笑"是 EXPLICIT（幻象文本直接描述了），和 W₀ 里"椅子刮地板"的 EXPLICIT 没有区别。

世界之间的区别不在置信度，在**本体论层级**：
- W₀ 是"叙事现实"——渲染器应该用完整的视觉保真度
- W₁/W₂/W₃ 是"角色主观体验"——渲染器可以用不同的视觉风格（模糊、色偏、快速剪辑）

但这种视觉风格的区分是**渲染器的表现层决策**，不是 LLEE 的世界状态。LLEE 只需要标注"这个世界状态属于哪个层级"。

## 四、世界间的证据交换

更有趣的是世界之间的信息流动：

```
W₃ → W₀ 的证据传递：
  Karl 在 W₃ 中"看到"Ashley 的反应（好奇）
  → 这成为 W₀ 中 Karl 决策的依据
  → 但 W₃ 的 Ashley 反应不是 W₀ 的事实
  → 它是 PREDICTED 级别的证据（来自角色的预知能力）

W₀ → W₁/W₂/W₃ 的证据传递：
  W₀ 中的教室布局、同学位置 → 被继承到所有幻象世界
  → 这是 CONTEXT 级别的证据（从主世界继承）
```

这种"世界间证据交换"和 WSM 的"跨段落状态继承"是同一个机制的不同尺度：
- 段落间继承：同一世界内，前一段的状态传递到后一段（衰减 0.9）
- 世界间继承：主世界的状态传递到子世界（完全继承，但标注为 CONTEXT）
- 世界间反馈：子世界的结果传递回主世界（标注为 PREDICTED，置信度上限 0.4）

## 五、叙事学中的对应物

这个多世界结构在叙事学中有成熟的理论框架：

- **嵌套叙事（embedded narrative）**：故事中的故事。《一千零一夜》的框架结构。
- **可能世界语义学（possible worlds semantics）**：Marie-Laure Ryan 的叙事理论。每个叙事都定义了一个"文本实际世界"（TAW）和多个"文本替代世界"（TAP）。
- **心理叙事学（psychonarratology）**：角色的信念、欲望、意图各自构成一个"心理世界"。

LLEE 的多世界扩展可以直接映射到 Ryan 的框架：

```
TAW (Textual Actual World) = W₀ = 主世界
TAP (Textual Alternative Possible worlds):
  K-world (Knowledge) = 角色所知的世界
  W-world (Wish) = 角色期望的世界
  O-world (Obligation) = 角色应该做的世界
  F-world (Fantasy/Forecast) = Karl 的 Vision A/B/C
```

## 六、对 LLEE 定位的深化

这个思考确实在深化 LLEE 的定位。

当前定位（v0.5）：**LLEE 是叙事文本到渲染参数的中间表示。**

深化后的定位：**LLEE 是叙事世界的声明式描述语言，支持多层世界的嵌套、继承和证据交换。**

区别在于：
- 前者把 LLEE 定位为一个"翻译工具"（文本 → 参数）
- 后者把 LLEE 定位为一个"世界建模语言"（可以描述任意复杂的叙事本体论）

后者的学术价值更高，因为它连接到了可能世界语义学、心理叙事学、甚至模态逻辑。但工程复杂度也更高。

**建议：Phase 0-3 保持当前的单世界架构。在论文中用一节 Discussion 描述多世界扩展的理论框架，用 Karl 的 Vision A/B/C 作为具体案例。Phase 4+ 实现 WorldStack。**

## 七、对计划的具体建议

### 立即可做（论文层面）
1. 在论文 Introduction 或 Design Principles 中明确声明：
   "LLEE 的置信度衡量文本证据强度，不衡量物理真实性。
    架空世界和真实世界在 LLEE 框架内享有平等的本体论地位。"

2. 在论文 Discussion 中用 Karl 的 Vision A/B/C 作为案例，
   描述多世界扩展的理论框架（WorldStack + EvidenceBridge）。

### Phase 4+ 可做（工程层面）
3. 实现 WorldStack：多层世界状态的嵌套管理
4. 实现 EvidenceBridge：世界间的证据传递规则
5. 渲染器支持：不同世界层级使用不同的视觉风格

### 不需要改变的
- 当前 schema 不需要改——单世界 WorldState 是多世界 WorldStack 的特例
- 当前衰减规则不需要改——世界内的衰减逻辑不受多世界影响
- 当前 Parser prompt 不需要改——Parser 一次只处理一个段落，段落属于哪个世界是元数据
