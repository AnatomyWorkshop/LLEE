# LLEE 开发笔记 — 数学模型联想与 v0.7 形式化路线图

> 2026-04-23

## 一、Doubao 数学模型文档的核心内容

文档分两部分：

**Part 1：混沌理论与不可预测性**
- 李雅普诺夫指数 λ：描述误差指数发散的标准工具
- 雅可比矩阵 J = ∂F/∂x：偏导数矩阵，特征值实部 > 0 → 不可预测
- 量子力学的不可预测：波函数演化确定（薛定谔方程），测量结果本质随机（玻恩规则）
- 关键反直觉结论：模型复杂度 ≠ 不可预测程度（Logistic Map 极简但极混沌）

**Part 2：世界模型中的预测难度量化**
- 转移熵 TE(s_{t+1} | s_t)：当前状态能减少多少下一状态的不确定性
- 条件互信息 I(s_{t+1}; s_t)：两状态间的信息量
- 局部李雅普诺夫指数 λ_local(s_t)：对每个具体状态计算预测难度
- 预测不确定性分解：H_epistemic（模型不懂）+ H_aleatoric（世界本身随机）

## 二、LLEE 当前版本（v0.6）与这些数学工具的关系

**直接回答：v0.6 没有引入任何这些数学工具。**

LLEE 的 WSM 是静力学的，不是动力学的：
- 它接收 Parser 从文本中提取的"事实"，然后应用这些事实
- 它从不尝试预测"下一个世界状态可能是什么"
- 它的"不可预测性处理"是回避（标记 UNDEFINED），而非量化

| 世界模型（Dreamer/Sora） | LLEE v0.6 |
|---|---|
| 学习 s_{t+1} = F(s_t, a_t) | 不学习任何转移函数 |
| 计算 λ_local、条件熵 | 不计算预测难度 |
| 在不确定区域放弃长规划 | 在不确定区域标记 UNDEFINED |
| 处理动力学 | 处理静力学 |

这不是缺陷，是设计选择。LLEE 的信任边界哲学是：**宁可遗漏，不可注入**。UNDEFINED 是这个哲学的实现，不是数学上的妥协。

## 三、三条自然联想路径（v0.7 的数学化方向）

### 路径 1：衰减率 → 条件互信息的数据驱动估计

当前 LLEE 的衰减机制：
```
ATMOSPHERIC: decay_rate = 0.7
CONTEXTUAL:  decay_rate = 0.9
```

这两个数字是手工设定的启发式值。它们的信息论等价物是：

```
decay_rate ≈ I(s_{t+1}; s_t) / I(s_t; s_t)
           = 当前状态对下一状态的条件互信息 / 自信息
```

**v0.7 的形式化路线**：
1. 用 Phase 1-2 积累的段落序列数据，估计每种证据类型的条件互信息
2. 用估计值替换手工衰减率
3. 这将把"为什么 ATMOSPHERIC 衰减更快"从"直觉"变成"数据"

具体地，对于证据类型 e，其衰减率应满足：
```
decay_rate(e) = exp(-H(s_{t+1}^e | s_t^e))
```
其中 H 是条件熵，s^e 是状态中与证据类型 e 相关的维度。

### 路径 2：置信度 → 预测不确定性的双分量分解

当前 LLEE 的置信度是单值标量（0.0-1.0），混合了两种本质不同的不确定性：

```
confidence = 0.3  可能意味着：
  (a) 文本证据弱（认知不确定性：模型没有足够信息）
  (b) 叙事本身模糊（偶然不确定性：作者故意不确定）
```

**v0.7 的形式化路线**：将置信度分解为双分量：

```python
class Evidence(BaseModel):
    confidence: float          # 总置信度（向后兼容）
    epistemic: float = 0.0     # 认知不确定性：文本证据不足
    aleatoric: float = 0.0     # 偶然不确定性：叙事本身随机/模糊
```

物理意义：
- epistemic 高 → 需要更多文本证据，Parser 应该继续搜索
- aleatoric 高 → 叙事本身未确定，标记 UNDEFINED，渲染器用中性默认值
- 这将使 UNDEFINED 机制获得信息论基础

IEI 的根源可以用这个框架精确描述：
> LLM 将本应属于 aleatoric 不确定性（叙事世界本身未确定的情感）错误地当作 epistemic 不确定性（模型应该推断出来的情感）来处理，于是用统计先验强行填充。

### 路径 3：Phase 4 预测器 → 局部李雅普诺夫指数的叙事意义

Phase 4 计划训练一个上下文预测器：
```
输入：WSM 状态向量 + 段落嵌入
输出：下一状态的属性概率分布
```

一旦有了这个预测器 F，就可以计算：
```
J(s_t) = ∂F/∂s_t    # 局部雅可比矩阵
λ_local(s_t) = max Re(Eig(J(s_t)))
```

**叙事意义**：
- λ_local > 0 的叙事节点：微小的文本差异导致完全不同的下一状态预测
  → 这是叙事的"分岔点"（narrative bifurcation）
  → 系统应该在这里放弃预测，退回 UNDEFINED，或请求作者介入
- λ_local < 0 的叙事节点：状态稳定，预测可靠
  → 这是叙事的"惯性段"（narrative inertia）
  → 系统可以做较长的状态继承

这将把 LLEE 的"叙事节奏感"从直觉变成可计算的量。

## 四、v0.7 论文需要的数学模型清单

按优先级排序：

### 必须有（核心贡献的数学化）

**M1：WSM 的形式化定义**
```
W = (E, V, S, O, H, T, N)    # 世界状态
Δ = (Δ_E, Δ_V, ..., Δ_N)    # 差分编码
W_{t+1} = WSM(W_t, Δ_t)     # 状态转移（确定性，非预测性）
```

**M2：证据置信度的衰减方程**
```
c_{t+1}(e) = decay_rate(level(e)) × c_t(e)
```
加上 narrative_break 的分段处理：
```
c_{t+1}(ATMOSPHERIC) = 0                    if narrative_break
c_{t+1}(CONTEXTUAL)  = 0.5 × c_t(CONTEXTUAL)  if narrative_break
```

**M3：置信度天花板的形式化**
```
c_t(e) ≤ ceiling(level(e))    ∀t, ∀e
```
这是 Zero-Introspection 原则的数学表达：证据强度上限由证据类型决定，不由内容决定。

**M4：IEI 的形式化定义**
```
IEI(Δ, T) = 1  iff  ∃e ∈ Δ.emotion_updates :
    e.emotion ≠ UNDEFINED  ∧  T contains no emotional evidence for e.entity
```
其中 T 是源文本，"contains no emotional evidence" 需要操作化定义（中性段落测试）。

### 应该有（支撑核心论点的数学化）

**M5：填充率的形式化**
```
fill_rate(Δ) = |{f ∈ fields(Δ) : Δ[f] ≠ None}| / |fields(Δ)|
```
按证据类型分组：
```
fill_rate_env(Δ) = |{f ∈ env_fields(Δ) : Δ[f] ≠ None}| / |env_fields(Δ)|
```

**M6：证据校准度（Cohen's Kappa）**
```
κ = (P_o - P_e) / (1 - P_e)
```
其中 P_o 是 Parser 与人工标注的一致率，P_e 是随机一致率。

**M7：衰减率的信息论解释（联想路径 1 的初步形式化）**
不需要完整推导，只需要在 Discussion 中写：
```
decay_rate(e) ≈ I(s_{t+1}^e; s_t^e) / H(s_t^e)
```
并说明当前手工值（0.7, 0.9）是对这个比值的近似，Phase 4 将用数据估计。

### 可以有（增强理论深度，但不是必须）

**M8：预测不确定性分解（联想路径 2 的初步形式化）**
```
H_pred(s_t) = H_epistemic(s_t) + H_aleatoric(s_t)
```
在 Future Work 中提出，不需要实现。

**M9：叙事分岔点的局部李雅普诺夫指数（联想路径 3）**
在 Phase 4 Future Work 中提出，作为预测器实验的延伸。

## 五、一个重要的认识论区分

Doubao 文档描述的数学工具（李雅普诺夫指数、转移熵）是为**动力学系统**设计的——它们描述"系统如何随时间演化"。

LLEE 是一个**静力学系统**——它描述"文本在某一时刻声明了什么"。

这个区分意味着：

1. **不能直接套用**这些工具到 LLEE 的当前架构
2. **可以类比**这些工具来理解 LLEE 的设计选择
3. **只有在 Phase 4 引入预测器之后**，这些工具才真正适用

v0.7 论文的数学化，应该从 M1-M4（静力学形式化）开始，而不是直接跳到李雅普诺夫指数。后者是 v0.8 或 Phase 4 论文的内容。

## 六、联想：LLEE 的"量子测量"类比

Doubao 文档提到量子力学的两层不可预测性：
- 波函数演化：确定（薛定谔方程）
- 测量结果：本质随机（玻恩规则）

这和 LLEE 的架构有一个结构上的相似：
- WSM 状态演化：确定（apply_delta 是纯函数）
- 渲染器输出：可以是随机的（渲染器可以在 UNDEFINED 区域自由发挥）

更深的类比：
- 波函数 ψ ↔ 证据栈（evidence stack，v2 设想）
- 测量 ↔ 渲染器"观测"证据栈
- 坍缩 ↔ 渲染器从矛盾证据中选择一个渲染方向

这不是比喻，是同一个数学结构：**概率分布的叠加，观测导致坍缩**。

当前 v0.6 的 emotion 是单值覆盖（已坍缩的状态），证据栈是未坍缩的状态。v0.7 如果实现证据栈，就真正进入了这个数学结构。

---

## 七、给 v0.7 论文的一句话建议

> v0.7 的数学化不是为了"看起来更严谨"，而是为了让每一个设计决策都有一个可以被质疑、被测试、被改进的数学表达。从 M1-M4 开始，把当前已经实现的机制形式化；然后用 M5-M7 支撑实验数据；最后用 M8-M9 指向未来。这是从"工程约束层"走向"叙事世界模型理论"的路径。
