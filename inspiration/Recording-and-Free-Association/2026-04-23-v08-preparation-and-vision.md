# LLEE 开发笔记 — 从约束层回到世界引擎：0.8 论文的必要准备

> 2026-04-23

---

## 一、v0.7 的战术价值与战略局限

v0.7 证明了一件事：**结构化约束可以在不同 LLM 上实现零 IEI。** 这是必要的基础工作。但如果 LLEE 的故事止步于此，它就只是一个 NLP 安全插件——有用，但不值得为之建立一套理论体系。

v0.7 的定位——"约束层，不是生成系统"——是对 GLM 批评（"数学裸奔"）和 Constantly-review 质疑（"LLEE 描述不了画面"）的防御性回应。防御是必要的，但防御不是目的。

**终极目的从未改变：用语言（及其扩展）构建一个可感知、可修改、可信赖的虚拟世界。**

问题是：当前的工作是这个终极目的的必要 subset 吗？

---

## 二、当前工作与终极目的的关系

### 终极目的的分解

```
终极目的：一人一代宇宙
  ├─ 感知层：从任意信息源（文本、视觉、音频、用户交互）提取世界状态
  ├─ 信念层：融合、衰减、审计来自多源的证据，维护一致的世界信念
  ├─ 生成层：从世界信念生成可体验的视听触内容
  └─ 交互层：用户修改 → 信念更新 → 重新生成 → 因果一致
```

### 当前工作覆盖了什么

```
v0.6-v0.7 覆盖：
  ✅ 感知层的文本子集（Parser + 证据分级）
  ✅ 信念层的核心机制（WSM + 衰减 + 天花板 + 叙事断裂）
  ✅ IEI 抑制（信任边界的工程验证）
  ✅ 数学框架的初步形式化（M1-M7）
  
  ❌ 感知层的多模态扩展
  ❌ 信念层的证据栈（矛盾证据共存）
  ❌ 信念层的贝叶斯更新（回溯修正）
  ❌ 生成层（渲染适配器只有设计，没有实现）
  ❌ 交互层（完全未触及）
```

### 结论：当前工作是必要的 subset

是的。但它是 subset 中最保守的部分——证明"不会出错"。终极目的还需要证明"能做对事"。

---

## 三、IEI 不应再是切入点

v0.7 用 IEI 作为论文的核心叙事线索。这在 v0.7 是合理的——它是最容易量化、最容易跨模型验证的指标。

但 IEI 作为切入点有一个致命问题：**它把 LLEE 定义为一个"防止坏事发生"的系统，而不是一个"让好事发生"的系统。**

0.8 论文的切入点应该是：**世界状态的可信构建与演化。** IEI 抑制是这个更大故事中的一个已解决的子问题，不是故事本身。

---

## 四、0.8 论文需要的核心转变

### 从"约束层"到"世界信念引擎"

| v0.7 的语言 | 0.8 应该说的 |
|---|---|
| LLEE 是约束层 | LLEE 是世界信念引擎，约束是其核心能力之一 |
| LLEE 不生成内容 | LLEE 维护世界信念，渲染器从信念中生成内容 |
| UNDEFINED 是诚实的沉默 | UNDEFINED 是主动的证据请求触发器 |
| 静态声明系统 | 信念演化系统（声明是演化的快照） |
| 渲染是别人的事 | 渲染是信念的物理投影，LLEE 定义投影规则 |

### 数学骨架的升级方向

v0.7 的数学是"参数的形式化"——给已有机制写方程。0.8 需要"架构的形式化"——用数学定义系统能做什么、不能做什么。

**需要引入的数学工具**：

1. **置信度的多因子分解**：
   $$c(e) = \rho_{\text{source}}(e) \cdot \delta_{\text{context}}(e) \cdot \phi_{\text{freshness}}(e)$$
   - $\rho$：来源可靠性先验（传感器/解析器的固有噪声）
   - $\delta$：语义偏离度（证据与当前上下文的一致性）
   - $\phi$：新鲜度（时间衰减的泛化）
   
   当前的单值置信度是这三个因子的乘积。拆开后可以分别审计"为什么不信任这条证据"。

2. **证据栈的概率单纯形表示**：
   对于属性 $a$，其证据栈 $\mathcal{E}(a) = \{e_1, ..., e_k\}$ 张成一个概率单纯形：
   $$P(a = v) = \sum_{e_i \in \mathcal{E}(a)} w_i \cdot \mathbf{1}[e_i.\text{value} = v]$$
   其中 $w_i = c(e_i) / \sum_j c(e_j)$。渲染器从这个分布中采样或取期望。

3. **信任边界的几何解释**：
   世界状态空间中，LLEE 定义了一个"可信区域" $\mathcal{T} \subset \mathcal{W}$。任何渲染结果必须落在 $\mathcal{T}$ 内。$\mathcal{T}$ 的边界由证据天花板和衰减率共同决定。IEI 就是渲染结果落在 $\mathcal{T}$ 之外。

4. **衰减率的信息论基础**（从 v0.7 M7 升级）：
   不再是 Discussion 中的类比，而是核心框架的一部分：
   $$r_\ell = \exp\left(-\frac{H(s_{t+1}^\ell \mid s_t^\ell)}{H_{\max}}\right)$$
   归一化后，衰减率直接由证据类型子空间的条件熵决定。

## 五、渲染仍然是 LLEE 的重点

stand-up-again.md 说得对：如果 LLEE 只是约束层，渲染就是"别人的事"。但如果 LLEE 是世界信念引擎，渲染就是"信念的物理投影"——LLEE 必须定义投影规则。

这意味着 0.8 论文需要：

1. **渲染映射的物理锚点**：不是无单位的 render_intensity，而是每种证据类型到具体渲染参数空间的映射：
   - 情绪证据 → BlendShape 权重范围
   - 环境证据 → 色温范围、雾效密度范围
   - 物理证据 → 力/扭矩范围
   
2. **证据栈的渲染语义**：矛盾证据不是 bug，是潜台词的工程实现。面部通道取 EXPLICIT 证据，声音通道取 BEHAVIORAL 证据 → 微笑 + 颤抖 = 潜台词。

3. **UNDEFINED 的渲染语义升级**：从"用默认值"到"触发证据采集"。当关键属性为 UNDEFINED 时，系统可以：
   - 向相邻段落的证据栈插值（低强度）
   - 向用户请求澄清
   - 向其他模态传感器请求补充

## 六、按当前计划走能实现终极目的吗？

**能，但需要在 Phase 2-3 之后做一次方向性调整。**

当前计划的 Phase 1-3 是必要的：
- Phase 1 证明了约束有效（IEI=0）
- Phase 2 将证明渲染可行（Three.js 适配器）
- Phase 3 将证明人类认可（认知相关性）

但 Phase 4 需要从"进化实验"扩展为"架构升级"：
- 证据栈实现（从单值到多值）
- 置信度多因子分解（从单标量到三因子）
- 渲染映射物理锚点（从无单位到有物理语义）
- 多模态证据源接入（从文本到视觉/音频）

这些不是"远期愿景"，是 0.8 论文必须至少在设计层面完成的工作。

## 七、0.8 论文的切入点草案

不再用 IEI 切入。用这个：

> 构建虚拟世界需要回答一个根本问题：系统应该相信什么？当文本说"洞穴"，视觉模型看到"岩壁"，物理引擎计算出"回声"，用户说"让它更暗"——这四条来自不同源头、不同可靠性、可能相互矛盾的信息，如何融合为一个一致的世界信念？
>
> LLEE 是一个世界信念引擎。它不生成世界的任何部分；它维护一个关于世界的、可审计的、多源融合的信念状态。渲染器从这个信念中生成视听内容。用户修改信念，世界随之改变。
>
> 本文的贡献是：(1) 一个多因子置信度框架，将证据可靠性、语义一致性和时间新鲜度分离为独立可审计的因子；(2) 证据栈机制，允许矛盾证据共存并由不同渲染通道分别消费；(3) 从文本约束到多模态信念融合的架构扩展，证明核心机制（衰减、天花板、IEI 检测）在多源场景下无需修改即可工作。
